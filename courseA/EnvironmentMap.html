【Map-環境構築とテスト】

GitHub Repository：

こちら です．

クローンしてお手元の開発環境でビルドし，動作環境を行ってください．

プロジェクトのインポート方法：

  • eclipse：
    1. 上部メニューからWindow>Perspective>Open Perspective>Other>Git を選択。
    2. 画面左の「Git Repository」の上部メニューからGit Cloneを選択。
    3. Clone 完了後、Clone したプロジェクト（今回はMap）を右クリックし、「Import Projects...」を選択。
    4. インポート完了後、「Package Explorer」から該当のプロジェクトを選び右クリック。
    5. メニューから「Configure>Add Gradle Nature」を選択すれば完了です。

  • IntelliJ IDEA：
    1. 上部メニューからFile>New>Project From Version Control... を選択。
    2. URIに GitHubのClone URIを入力。
    3. Clone 完了後、自動で Build が実行されたら完了です。

テストコードについて：

【Map】のテストコードは /src/test/java/以下にあります．

  • TestGPSUpdate.java： 設計変更対象の動作確認用のテストコードです。
  • TestUpdateManually.java： 設計変更後、機能追加を行った後のテストコードです。

実行方法：

  • eclipse： 対象のテストコードを「Gradle Test」として実行してください。
  • IntelliJ IDEA： 対象のテストコードを開き、実行ボタンをクリックして実行してください。
