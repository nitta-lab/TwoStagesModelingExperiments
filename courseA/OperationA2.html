【SSDStore】
ツール課題

課題概要：

先ほどの課題では，機能追加を容易にするリファクタリングを手作業で行っていただきました．本課題では，本研究室で提案しているビジュアルモデリングツールを用いて， 先ほどのリファクタリングと同等の作業をツールの GUI 上で行っていただきます． 本課題は，所定のブランチへの commit & push および 作業完了後アンケートにご回答いただくことで完了となります． 本モデリングツール，および作業手順を以下で説明します．

ビジュアルモデリングツール：

ビジュアルモデリングツールは，ソフトウェアの仕様と深くかかわるデータの流れを最初に決定して，パフォーマンスや可変性などの性能に関わる部分を GUI 上で後から自由に設計できるようにしたモデリングツールです． 設計は主に，データ転送方式としてPUSH型を選ぶかPULL型を選ぶかによって決定していきます．設計完了後は，実行可能な Java のプロトタイプを自動生成することが可能です．

DLリンク：

ビジュアルモデリングツール（Jarファイル）は， こちら からDLしてください．
java -jar VisualModelingTool.jar
で実行することができます．JRE のバージョン 1.8 以上に対応しています．

ツールの操作方法：

操作方法はこちらからご参照ください．
本ツールを使用する前に一通り手順，サンプルプログラム，UMLをご確認いただき， 操作方法を理解した上で課題を進めていただきます．

ツール作業内容：

はじめに：

作業に着手する前に，mainブランチから，メッセージでお伝えした自分専用のブランチ (user??Generated)を新規に作成し，ブランチのpushを行ってください．
本課題では本ツールによって出力されたJavaプロトタイプのソースコードを用いて，【SSDStore】のテストプログラムが正常に動作するまでは作業完了とは見なされませんのでご注意ください．

ビジュアルモデリングツールでの作業：

はじめに，こちらから【SSDStore】プログラムのモデルファイルをDLしていただきます．
DLしていただいたモデルファイルは，本ツールを起動し読み込んでいただきます．操作方法については上記のビジュアルモデリングツールの項をご参照ください．
モデルファイルを読み込んだ後，本ツール右下のナビゲーションウィンドウがPUSH/PULLの選択になっていることを確認してください．
確認ができたら，PriceとItemsByPrice，SiteAとItemsByPrice,ItemsByCapacity，CapacityとItemsByCapacity間のデータ転送方式をPULLに変更してください．
変更が完了しましたら，右下のナビゲーションウィンドウからPUSH/PULLの拡張を選択していただきます．
