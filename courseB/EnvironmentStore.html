【SSDStore】
環境構築とテスト

GitHub Repository：

GitHub のリポジトリから clone してお手元の開発環境でビルドし，テストケースを実行して動作確認を行ってください．
SSDStore のリポジトリは こちら (別タブで開きます) です．
main ブランチから clone してください．

プロジェクトのインポート方法：

  • eclipse：
    1. 上部メニューから「Window>Perspective>Open Perspective>Other>Git」 を選択．
    2. 画面左の「Git Repository」の上部メニューから「Git Clone」を選択．
    3. clone 完了後，clone したプロジェクト（今回はMap）を右クリックし，「Import Projects...」を選択．
    4. インポート完了後，「Package Explorer」から該当のプロジェクトを選び右クリック．
    5. メニューから「Configure>Add Gradle Nature」を選択すれば完了です．
  • IntelliJ IDEA：
    1. 上部メニューから「File>New>Project From Version Control...」 を選択．
    2. URIに GitHubのClone URIを入力．
    3. clone 完了後，自動で Build が実行されたら完了です．

テストコードについて：

【SSDStore】のテストコードは /src/test/java/以下にあります．

  1. TestSSDStoreUpdate.java： リファクタリング対象の動作確認用のテストコードです．
  2. TestMultiSSDStoreUpdate.java： 機能追加後のテストコードです．

なお main ブランチにはテスト(1)が，refactored ブランチにはテスト(1), (2)があります．

実行方法：

  • eclipse： 対象のテストコードを「Gradle Test」として実行してください．
  • IntelliJ IDEA： 対象のテストコードを開き，実行ボタンをクリックして実行してください．
