【実験の主旨及び流れ】

甲南大学知能情報学部のソフトウェア設計検証研究室です．
この度は私共の実験にご協力頂きまして誠にありがとうございます．

実験の主旨と概要：

Javaによる実務経験3年以上のエンジニアの方々を対象に，被験者として実験へのご協力をお願いいたします．
この実験は，本研究室で開発したビジュアルモデリングツールの設計変更作業における有効性の評価を目的としています．
設計変更の対象となるJavaプログラムは2つあり，それぞれのプログラムに対してリファクタリング作業，モデリングツールの操作，機能追加作業に取り組んでいただきます．
すべての作業はお手元のPC上で行っていただき，課題を終えるたびに，アンケートにお答えいただきます．
全体の作業時間としては，3~5時間程度を想定しています． これらの課題はすべてお手元のPC上で手作業で行っていただきます．課題を終えるたびにGitHubに commit ＆ push をしていただいて，アンケートにお答えいただきます．
お答えいただくアンケートは最初と最後，各リファクタリング課題，モデリングツール課題，機能追加課題の完了後の計8個あります．
アンケートへの回答が終わるたびに作業を中断していただいて構いません． 作業はすべてWebおよびローカルのPC上で完結します．

※過去に実施した私共のいずれの実験にご協力いただいた方でも，こちらの実験には問題なくご参加いただけます．

実験における作業内容について：

対象となるJavaプログラムはいずれも小規模なアプリケーションです．それぞれのプログラムに対して，リファクタリング課題，モデリングツール課題，機能追加課題に取り組んでいただきます．
各Javaプログラムには機能追加の課題が用意されておりますが，いずれのJavaプログラムも，最初は機能追加が容易にできないような設計になっています．
そこで，まず機能追加が容易になるようJavaプログラムのリファクタリングを手作業で行っていただきます．
また，本モデリングツールでは，GUI上で設計変更を行うことが可能であるため，手作業で行ったのと同様のリファクタリングをツール上でも行っていただきます．（モデリングツールの解説は実験の中で行います．）
最後に，ツールから出力された設計変更後のJavaプロトタイプに対して，機能を追加する作業を行っていただきます．
実験では，まずGitHub上に公開した実験用のJavaプログラムを，お手元のPCにダウンロード（clone）していただき，動作確認を行った上で，手作業でリファクタリングを行っていただきます．
いくつかの課題では，お手元の時計で作業に要した時間を計測していただきます．また，各作業の結果はGitHubの作業用ブランチに課題ごとに commit ＆ push していただきます．
ビジュアルモデリングツールは，WebからJarファイルをダウンロードしてお使いいただきます．
開発環境は何をお使いいただいても結構ですが，主に eclipse と IntelliJ IDEA を想定しています．それぞれの課題を終えるたびに，アンケートにお答えいただきます．
また，実験の開始前と終了後にも1つずつアンケートにお答えいただきます．

ご応募までの手順：

  1. 本HTML(実験に関する説明，個人情報の取り扱いに関する説明，報酬を含む契約に関する説明)をお読みいただきます．
  2. 1の全てに同意できる場合は，下記の同意確認フォームにその旨を入力していただきます．
  3. ご同意をもって応募の条件とさせていただきます．ご同意いただけない場合はご応募いただけません．

実験全体の流れ：

  1. 下記の同意確認フォームにご記入いただきます．
  2. 実験にご同意いただける場合は，個人アンケートにお答えいただきます．
  3. 実験へのご同意，個人アンケートの内容，プロフィールの内容等を元に実験依頼の可否を判断させていただきます．
  4. 実験依頼の可否についてご連絡いたします．実験をご依頼する場合は，実験準備が整い次第，実験用のWebページをこちらからお伝えいたします．(実験準備に数日を要する可能性があります．)
  5. 実験に取り組む上で必要となる基本概念について解説します．
  6. 1つめの実験用プログラムを手作業でリファクタリングしていただきます．（時間計測が必要です．）
    • 実験用プログラムの仕様と設計を理解していただきます．
    • GitHub上のプログラムを clone していただき，お手元の環境でビルドと動作確認を行っていただきます．
    • リファクタリング課題の目的と手順を説明いたします．
    • 作業用のブランチを切っていただき，時間計測をしながらリファクタリング作業を行っていただきます．
    • 作業の結果をブランチに commit ＆ push していただきます．
    • リファクタリング課題アンケートにお答えいただきます．
  7. 次にリファクタリングで行った設計変更と同等の変更をモデリングツールのGUI上で行っていただきます．（時間計測は不要です．）
    • WebからモデリングツールのJarファイルをダウンロードしていただきます．
    • モデリングツールを使用する上での概念と操作方法について説明をいたします．
    • モデリングツール上で，モデルファイル（ツールの専用のソースファイル）を読み込んでいただきます．
      このモデルファイルは，6.のリファクタリングを行う前のJavaプログラムの設計に相当します．
    • モデリングツールのGUI上で，読み込んだモデルの編集作業を行っていただきます．この編集作業は上記のリファクタリングに相当します．
    • モデリングツールのコード生成機能を使って，編集後のモデルをJavaプログラムとして出力していただきます．
      出力されたプログラムは，6.でリファクタリングを行った後のプログラムとほぼ同じものになります．
    • 出力されたJavaプログラムをブランチに commit ＆ push していただきます．
    • ツール課題アンケートにお答えいただきます．
  8. 7.で出力されたプログラムに対して，さらに機能追加作業を行っていただきます．（時間計測が必要です．）
    • 作業の目的と手順を説明いたします．
    • 機能追加作業を行っていただき，作業が完了したことをテストケースを使って確認していただきます．
    • 作業の結果をブランチに commit & push していただきます．
    • 機能追加課題アンケートにお答えいただきます．
  9. 2つめの実験用プログラムについても同様に手作業でリファクタリングしていただきます．（時間計測が必要です．）
  10. 2つめの実験用プログラムのモデルに対してモデリングツール上で編集作業を行っていただきます．（時間計測は不要です．）
  11. 10.で出力されたプログラムに対して，さらに機能追加作業を行っていただきます．（時間計測が必要です．）
  12. 最後に実験全体に関してのアンケートにお答えいただきます．

以上が実験の流れです．
全ての作業はWebおよびローカルのPC上で完結します．

同意事項：

実験中の危険性に関して説明いたします．
通常のPC画面に向かっている際と危険性は変わりません．

実験時間(拘束時間)に関して説明いたします．
全体の作業としては3~5時間を想定していますが，人によっては多少前後する可能性があります．
時間に余裕がある際に行ってください．
また，後日大学にお越しいただいたり，こちらから連絡を行うことはありません(参加者に不利益が発生する恐れのある場合は除く)．

実験への参加取りやめに関して説明いたします．
実験の途中で，参加を取りやめることが可能です．
その場合の報酬に関する取扱いついては．以下で説明いたします．

契約と報酬に関して説明いたします．

  • 報酬は作業全体に対して一律に(9,000円)支払われます．
  • 作業は全てのアンケートにお答えいただいた時点で終了とします．
  • 実験の途中で参加を取りやめた場合は，報酬は支払われません．あらかじめご了承下さい．
  • 契約及び報酬に関してご不明な点がございましたら，メッセージにてご連絡ください．
  • おひとりで，複数回ご担当することはできません．
  • 他の参加者に作業の情報を提供したこと，または，他の参加者からの情報をもとに作業を行ったことが判明した場合は，報酬の支払いをお断りする場合がございます．

ビジュアルモデリングツールでの操作履歴取得に関して説明いたします．
実験で使用するビジュアルモデリングツールは，ツール上での操作履歴を記録しログとして出力します．
さらに出力されたログを実験データの1つとして提出していただきます．ご提出いただいたログは，下記の個人情報として取り扱います．

個人情報の取り扱いについて説明いたします．

  • ご提供いただいた個人情報は研究目的以外では利用いたしません．
  • ご提供いただいた個人情報を第三者に提供・開示することはありません．
  • 研究成果として公開する場合は，統計処理を施し，個人の情報が特定できないようにしたデータのみを用います．
  • ご提供いただいた個人情報は不正アクセス，紛失，破壊，改ざん，漏洩等がないよう厳重に管理いたします．
